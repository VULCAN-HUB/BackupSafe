# 백업 세이프 (BackupSafe) — 설계 문서

- **프로젝트**: 6번 BackupSafe
- **작성일**: 2026-06-19
- **상태**: 확정 (구현 전)
- **스택**: Electron + Node.js (크로스플랫폼 단일 코드베이스)
- **저장 위치**: `D:\프로젝트\6_BackupSafe\`

## 1. 개요

사진가가 메모리카드 촬영본을 여러 백업 위치로 복사하고, SHA-256 해시 비교로 무결성을 검증한 뒤, 검증 통과 시 일괄 이름 변경까지 처리하는 크로스플랫폼 데스크탑 도구. 핵심 가치는 **백업 누락·손상에 대한 신뢰** — 문제를 절대 조용히 넘기지 않는다.

- **대상**: 메모리카드 백업 시 파일 누락·손상을 가장 두려워하는 사진가(업/취미).
- **플랫폼**: Windows / macOS (Electron). 코드 100% 공용, 드라이브 감지만 OS 분기.
- **포맷 기능 미포함** — "포맷해도 안전" 안내만, 실제 포맷은 사용자가 직접.

## 2. 확정된 결정 사항 (D1~D16)

| # | 항목 | 결정 |
|---|---|---|
| D1 | 이름 변경 시 메모리카드 원본 | **건드리지 않음.** 카드는 읽기 전용으로만 취급. 이름 변경은 백업 위치들에만 적용. |
| D2 | 복사 방식 | **파일 단위 순차 진행.** 각 파일은 카드에서 **1회만 읽어** 활성 백업 위치들에 팬아웃(fan-out) 기록. (위치를 통째로 1→2→3 순회하는 게 아님 — 그러면 카드를 위치 수만큼 읽게 되어 D3와 충돌.) 위치 간 병렬 쓰기는 추후 최적화. |
| D3 | 해시 검증 | 복사 중 읽은 바이트로 **원본 해시를 인라인 스트리밍 계산**(`hash.update(chunk)`, 청크 사이 이벤트 루프 양보로 메인 프로세스 거의 비차단)해 카드를 1회만 읽음. 검증 단계의 **대량 재해시는 `worker_threads` 오프로드 고려**(구현 시 판단). 진행률 표시. |
| D4 | 클라우드 동기화 폴더 | 로컬 폴더 기준으로만 해시 비교. "클라우드 서버 업로드 완료는 보장 못 함"을 안내 배지로 명시. |
| D5 | 화면 구조 | **단계별 마법사(Wizard)** 4단계. 검증 통과 게이트가 흐름과 자연스럽게 맞음. |
| D6 | 이름 변경 변수 토큰 | 커서 위치에 토큰을 **하나씩 삽입**, 자유 텍스트와 혼합. 예: `{날짜}_직접입력_{순번}`. |
| D7 | 날짜 형식 | 토글: 하이픈 포함(`2026-06-19`) / 미포함(`20260619`). |
| D8 | UI 프레임워크 | 순수 HTML/CSS/JS (React 미사용 — 단발성 마법사, 상태 단순). |
| D9 | `{순번}` 정렬 기준 | 기본 **촬영일시(EXIF) 오름차순** 1부터. EXIF 없으면 파일 수정일. 정렬 기준 토글(촬영일시/파일명) 제공. |
| D10 | "폴더명 변경" 범위 | **각 백업 위치 아래 최상위 결과 폴더 1개만** 대상. 카드 내부 하위 폴더 구조(`DCIM/100CANON` 등)는 변경하지 않음(부분 적용·충돌 사고 방지). |
| D11 | 디스크 여유 사전 체크 | 1차 = **단순 총량 비교**(카드 사용량 vs 각 위치 여유공간), 부족 시 Step2에서 경고. 위치별 정밀 예약·증분 계산은 향후(섹션 9). |
| D12 | 복사 목적지 레이아웃 | 각 백업 위치 **루트에 직접 투하하지 않고, 결과 폴더 1개를 생성**해 그 안에 복사(기존 파일 오염 방지). 기본 폴더명 = `{카드라벨}_{YYYY-MM-DD}`(카드 볼륨 라벨 없으면 `BackupSafe`로 폴백), Step2에서 편집 가능. **이 결과 폴더가 D10 "최상위 결과 폴더" = 폴더명 이름변경 대상.** 모든 위치에 동일한 결과 폴더명 사용. Step4 폴더 이름변경 시 Step2 기본값을 **덮어씀**. |
| D13 | 카드 내부 구조 | **보존**(평탄화 안 함). 카드의 하위 폴더 구조(`DCIM/100CANON/...`)를 결과 폴더 안에 그대로 재현. 상대경로 인덱스 기준으로 복사·검증. |
| D14 | CSV 리포트 인코딩 | **UTF-8 with BOM** 고정(Excel 한글 깨짐 방지). |
| D15 | 작업 중단 시 잔여 파일 | 카드 제거·취소 등 중단 시 **목적지 부분 파일을 자동 정리하지 않음**(원본 카드는 불변이라 안전). 검증이 누락/손상으로 노출, 사용자에게 재실행 권장. 자동 resume·정리는 향후(섹션 9). |
| D16 | 토큰의 파일/폴더 적용 구분 | **파일명 모드** = 전체 토큰 사용 가능. **폴더명 모드** = 파일 고유 토큰(`{순번}`·`{원본파일명}`) 비활성, `{날짜}`/`{클라이언트명}`/`{촬영지}` + 자유텍스트만 허용. 폴더의 `{날짜}` = 그 폴더 내 **최초 촬영일**(없으면 복사일). |

## 3. 아키텍처

**보안**: `contextIsolation: true`, `nodeIntegration: false`, preload + IPC. 렌더러는 파일시스템에 직접 접근하지 않고 메인 프로세스에 IPC로 위임.

```
main.js            앱 부트스트랩, 윈도우, IPC 라우팅
preload.js         contextBridge로 허용된 IPC 함수만 노출
services/
  driveScanner.js  이동식 드라이브 감지 (Win: PowerShell, Mac: /Volumes + diskutil) — process.platform 분기
  copier.js        파일 단위 복사 + 위치 팬아웃(결과 폴더 생성·구조 보존) + 진행률 이벤트, 실패 파일 기록
  hasher.js        SHA-256 인라인 스트리밍(복사 중 원본 해시 동시 계산) + 검증 재해시(worker_threads 고려)
  exif.js          EXIF 촬영일시 추출(exifr 의존성). 인덱스의 {EXIF촬영일} 채움, 없으면 파일 mtime 폴백
  verifier.js      위치별 정상/누락/손상 분류
  renamer.js       토큰 패턴 파싱 + 미리보기 + 충돌 감지 + 일괄 적용
  presets.js       백업 위치 프리셋 저장/불러오기 (userData JSON)
  notifier.js      OS 알림 (완료 / 손상·누락 강조)
renderer/
  index.html, app.js, steps/*  마법사 4단계 UI
```

- **메인 프로세스**: 파일시스템, 드라이브 감지, 복사, 해시, 알림, 폴더 다이얼로그.
- **렌더러 프로세스**: 마법사 UI, 진행률 표시. IPC로만 통신.
- **드라이브 감지**만 OS 분기, 동일 형식의 드라이브 목록을 반환해 나머지 코드는 플랫폼 무관.

## 4. 데이터 흐름 & 검증 로직

**1) 카드 선택 → 파일 인덱스**
재귀 탐색해 `{상대경로, 크기, mtime, EXIF촬영일}` 인덱스 생성(`exif.js`로 촬영일시 채움, 없으면 mtime 폴백). 복사·검증·이름변경의 기준.

**2) 복사 + 원본 해시 동시 계산 (카드 1회 읽기)**
각 활성 위치에 결과 폴더(`{카드라벨}_{날짜}`, D12) 생성 후:
```
각 파일마다(인덱스 상대경로 기준, 구조 보존 D13):
  카드에서 스트림 1회 읽기 → (a) 활성 위치들의 결과 폴더에 팬아웃 기록(하위 폴더 자동 생성)
                          → (b) 같은 바이트로 원본 SHA-256 누적(인라인 스트리밍)
  원본 해시를 인덱스에 저장
  진행률 이벤트 발행(파일 n/N, 누적 바이트, 남은 시간 추정)
실패 파일은 기록 → 검증에서 누락/실패로 노출
```

**3) 검증 — 위치별 분류**
```
각 백업 위치 × 각 파일:
  대상 존재? ──아니오→ 누락
  대상 SHA-256 == 원본 해시? ──아니오→ 손상
  일치 → 정상
```
위치별 집계 + 문제 파일 상세 목록. CSV/텍스트 리포트 내보내기(CSV는 **UTF-8 with BOM**, D14).

**4) 게이트 & 알림**
- 모든 활성 위치 전부 정상 → "포맷해도 안전" 안내 + 이름변경 활성화.
- 하나라도 누락/손상 → 빨간 경고 + 위치/파일 명시, 이름변경 잠금.
- OS 알림 2종: 완료(일반) / 손상·누락(강조).

**5) 이름 변경 (게이트 통과 후만)**
토큰 패턴 파싱 → 첫 3개 미리보기 → 모든 백업 위치에 동일 적용(원본 불변, D1). 폴더명/파일명 지원. 충돌 시 경고·중단. 확장자 보존.

## 6. 에러 처리 — "조용히 넘어가지 않는다"

- **복사 실패**(권한·디스크 꽉참·케이블 분리): 실패 목록 기록 후 계속 → 검증에서 누락 노출. 디스크 공간 부족은 복사 전 사전 체크.
- **해시 불일치(손상)**: 절대 정상 처리 안 함. 위치·파일 명시.
- **드라이브 감지 실패**: 빈 목록 대신 명확한 에러 + 새로고침 유도.
- **이름 변경 충돌**: 미리보기에서 차단, 부분 적용 금지.
- **카드 제거 감지**: 작업 중 카드 사라지면 즉시 중단·경고. 목적지 부분 파일은 자동 정리하지 않음(D15), 재실행 권장.
- 모든 사용자 대면 에러는 한국어, 무엇을/어디서/어떻게 대처 안내.

## 7. 테스트 전략 (TDD)

- **단위 테스트(주력, 메인 프로세스 순수 로직)**:
  - `hasher` — 알려진 바이트 SHA-256 정확성
  - `verifier` — 정상/누락/손상 3분류 (픽스처 디렉터리)
  - `renamer` — 토큰 파싱, 날짜 형식 토글, 순번 패딩, 정렬 기준(촬영일시/파일명), 파일/폴더 토큰 구분(D16), 폴더명 대상 범위, 충돌 감지, 미리보기
  - `exif` — EXIF 촬영일시 추출 + mtime 폴백(샘플 이미지 픽스처)
  - `copier` — 임시 폴더 복사 + 결과 폴더 생성 + 하위 구조 보존(D13) + 진행률 이벤트, 실패 파일 기록
  - `presets` — 저장/불러오기 라운드트립
  - `driveScanner` — 플랫폼 분기 (OS 명령 모킹)
- **통합**: 임시 카드 폴더 → 2개 백업 위치 복사 → 검증 → 이름변경 전체 플로우 1종.
- **수동 검증**: 실제 메모리카드로 오너 최종 확인. 렌더러 UI 헤드리스 한계로 핵심 로직은 메인 프로세스에 두고 단위 테스트 커버.

## 8. 빌드 & 크로스플랫폼

- 개발 중에는 `npm start`(`electron .`)로 즉시 실행 테스트. 패키징은 전체 기능 완성 후.
- `electron-builder`로 Windows `.exe`(현재 PC 빌드 가능), macOS `.dmg`/`.app`.
- **macOS 실빌드는 Mac 실기 도착 후** (Electron도 크로스컴파일·서명/공증은 실기 필요). 코드·빌드 설정은 지금 준비. cross-platform-prep 가이드라인과 동일 패턴.

## 9. 1차 범위 제외 (향후)
